## 2. 데이터 파이프라인 흐름 (수집 → 가공 → 적재)

```mermaid
flowchart LR
    subgraph Input["📂 입력 데이터"]
        A1["국토부 XLSX<br/>(17개 시도)"]
        A2["브랜드 FAQ<br/>(HTML/API/Selenium)"]
        A3["K Car 옥션 FAQ<br/>(Selenium)"]
        A4["위키피디아<br/>(requests+BS4)"]
    end

    subgraph Process["⚙️ Python 처리"]
        B1["prepare_data.py<br/>4축 비율 계산<br/>페르소나 코드 산출"]
        B2["crawl_brand_faq.py<br/>crawl_faq.py<br/>faq_common.py"]
        B3["crawl_car_images.py"]
        B4["setup_db.py<br/>load_to_mysql.py"]
    end

    subgraph Output["🗄️ MySQL (car_bti)"]
        C1[("region_stats<br/>17행")]
        C2[("company_faq<br/>750+건")]
        C3[("persona_cars<br/>64대 + 이미지 BLOB")]
    end

    A1 --> B1 -->|"region_stats.csv"| B4 --> C1
    A2 & A3 --> B2 --> C2
    A4 --> B3 --> C3
    B4 --> C2
    B4 --> C3
```

### 데이터 소스별 수집 방식

| 데이터 | 스크립트 | 방식 | 저장 테이블 |
|--------|----------|------|-------------|
| 지역 등록 통계 | `prepare_data.py` → `load_to_mysql.py` | XLSX → CSV → MySQL | `region_stats` |
| 브랜드 FAQ | `crawl_brand_faq.py` | requests / API / Selenium | `company_faq` |
| K Car 옥션 FAQ | `crawl_faq.py` | Selenium (탭 클릭) | `company_faq` |
| 차량 이미지 | `crawl_car_images.py` | requests + BeautifulSoup | `persona_cars` (LONGBLOB) |
| 차량·FAQ 시드 | `setup_db.py` | 초기 INSERT | `persona_cars`, `company_faq` |

## 4. AI 챗봇 RAG 아키텍처

```mermaid
flowchart TD
    Q["사용자 질문"] --> Intent["intents.py<br/>의도 분류"]
    Intent --> Search["retriever.py<br/>FAQ 유사도 검색"]

    Search --> Mode{"GEMINI_API_KEY<br/>설정 여부"}
    Mode -->|있음| Embed["gemini-embedding-001<br/>임베딩 검색"]
    Mode -->|없음| Keyword["키워드/n-gram<br/>폴백 검색"]

    Embed & Keyword --> Context["prompts.py<br/>[참고 FAQ] + 차량 + 지역 + 뉴스"]
    Context --> LLM{"AI 모드?"}
    LLM -->|🟢 AI| Gemini["gemini-2.5-flash<br/>근거 기반 답변"]
    LLM -->|🟡 기본| Rule["규칙 기반<br/>FAQ/추천 정리"]

    Gemini & Rule --> Answer["채팅 UI 출력<br/>chatbot/ui.py"]

    subgraph DB["MySQL"]
        FAQ[("company_faq")]
        Cars[("persona_cars")]
        Region[("region_stats")]
    end

    Search --> FAQ
    Context --> Cars
    Context --> Region
```

## 7. ERD (데이터베이스 관계 요약)

```mermaid
erDiagram
    region_stats {
        VARCHAR region PK "시도명"
        FLOAT eco_ratio "친환경 비율"
        FLOAT large_ratio "대형차 비율"
        FLOAT female_ratio "여성 비율"
        FLOAT import_ratio "수입차 비율"
        CHAR persona_code "4자리 Car-BTI"
    }

    persona_cars {
        INT car_id PK
        CHAR persona_code "매칭 유형"
        VARCHAR brand
        VARCHAR car_model
        VARCHAR price
        TEXT reason
        LONGBLOB img_data
        VARCHAR img_mime
    }

    company_faq {
        INT faq_id PK
        VARCHAR company
        VARCHAR car_category
        TEXT question
        TEXT answer
        VARCHAR persona_tags "E,G,L,... 태그"
    }

    region_stats ||--o{ persona_cars : "persona_code로 추천"
    persona_cars ||--o{ company_faq : "brand·persona_tags로 FAQ 매칭"
```

## 8. 발표용 한 줄 설명 (스크립트)

> **"Car-BTI는 국토부 등록 통계와 브랜드 FAQ를 MySQL에 모은 뒤, Streamlit으로 지역·성향·차량을 시각화하고, Gemini RAG 챗봇으로 근거 있는 상담까지 제공하는 통합 대시보드입니다."**

## 부록: 주요 파일 역할

| 구분 | 파일 | 역할 |
|------|------|------|
| UI | `app.py` | Streamlit 메인 (3탭) |
| UI | `chatbot/ui.py` | AI 챗봇 화면 |
| DB | `db_config.py` | MySQL 연결 |
| DB | `setup_db.py` | 테이블·시드 생성 |
| ETL | `prepare_data.py` | 국토부 XLSX → CSV |
| ETL | `load_to_mysql.py` | CSV → region_stats |
| 크롤링 | `crawler/crawl_brand_faq.py` | 브랜드 FAQ |
| 크롤링 | `crawler/crawl_faq.py` | K Car 옥션 FAQ |
| 크롤링 | `crawler/crawl_car_images.py` | 차량 이미지 |
| API | `news_api.py` | 네이버 뉴스 |
| AI | `chatbot/llm_client.py` | Gemini 연동 |
| AI | `chatbot/retriever.py` | FAQ RAG 검색 |
| AI | `chatbot/intents.py` | 의도·답변 생성 |
